# AI / GitHub Metrics

AI adoption and developer activity metrics extracted from GitHub pull requests and commits.

## Metrics

| Metric | Unit | Description |
|--------|------|-------------|
| **AI-assisted Pull Request Coverage** | percent | Share of merged PRs where the title, body, or commit messages contain AI-tool signals. |
| **AI Active Developers** | count | Number of team members who authored at least one merged PR with AI signals in the period. |

## How the Pipeline Works

The script `scripts/ai/pull-adoption-metrics.mjs` runs the following steps:

### 1. Build quarter windows

The script generates quarter boundaries (Q1–Q4) for each year from `AI_FROM_YEAR` (default: previous year) up to the current quarter. Each window is a date range, e.g. `2025-Q1` = `2025-01-01` to `2025-03-31`.

### 2. Fetch merged pull requests

For each quarter window, the script iterates over every repository in `backend/ai/config/github-repo-scope.json` and calls:

```
GET /repos/{owner}/{repo}/pulls?state=closed&sort=updated&direction=desc&per_page=100
```

It paginates until it finds PRs older than the quarter start date, then filters to only PRs where `merged_at` falls within the quarter window. PRs that were closed without merging are ignored.

### 3. Match PR authors to team members

Each merged PR's `user.login` (the GitHub username of the author) is matched against the roster in `backend/ai/identity/team-user-map.json`. If the author's `githubLogin` is not in the roster, the PR is skipped entirely — it does not count toward any team's totals.

If a team member has `activeFrom` or `activeTo` dates, the PR is only counted if the quarter window overlaps with their active period.

### 4. Scan for AI signals

For each matched PR, the script collects three text sources:

1. **PR title** (`pr.title`)
2. **PR body/description** (`pr.body`)
3. **All commit messages** — fetched via `GET /repos/{owner}/{repo}/pulls/{number}/commits`

These texts are concatenated and tested against a set of regex patterns. If **any** pattern matches, the PR is flagged as "AI-assisted".

### 5. AI signal patterns

```
/co-authored-by:.*copilot/i
/co-authored-by:.*\[bot\]/i
/co-authored-by:.*github/i
/\bcopilot\b/i
/\bai[- ]?(generated|assisted|created|authored)\b/i
/\bgenerated by\b/i
/\bcursor\b/i
/\bgithub copilot\b/i
/\bclaude\b/i
/\bchatgpt\b/i
/\baider\b/i
```

This is a heuristic approach. Common triggers include:
- GitHub Copilot's automatic `Co-authored-by: Copilot` commit trailer
- Developers mentioning "cursor" or "copilot" in PR descriptions or commit messages
- AI-generated commit messages containing phrases like "ai-generated" or "generated by"

### 6. Calculate metrics per team per quarter

For each team in each quarter:

- **AI-assisted Pull Request Coverage** = `(AI-flagged PRs / total matched PRs) × 100`
- **AI Active Developers** = count of unique GitHub logins that authored at least one AI-flagged PR

### 7. EDU portfolio rollup

The EDU row aggregates across all delivery teams using raw counts (not averaging percentages):

- **Coverage** = `total AI PRs across all teams / total PRs across all teams × 100`
- **AI Active Developers** = sum of AI-active developers across all teams

### Limitations

- **Heuristic detection**: The regex patterns may produce false positives (e.g. a repo named "cursor-utils") or false negatives (AI-assisted code with no textual signal).
- **Only merged PRs**: Draft PRs, open PRs, and closed-without-merge PRs are excluded.
- **Author-based matching**: If a team member uses a different GitHub account, their PRs will not be counted.
- **No code-level analysis**: The script does not inspect diffs or code content — it only reads PR metadata and commit messages.

If GitHub Copilot Metrics API becomes available for the org, the script should be updated to prefer that as the authoritative source.

## Repo Discovery

The discovery script scans configured GitHub orgs and maps repos to teams based on contributor activity:

```bash
node scripts/ai/discover-repos.mjs
```

This:

1. Lists all repos from `vancopayments` and `SmartTuition`
2. Filters to repos with activity in the last 6 months
3. Fetches contributors for each active repo
4. Matches contributor logins against `team-user-map.json`
5. Writes `backend/ai/generated/repo-team-mapping.json` with the full mapping
6. Updates `backend/ai/config/github-repo-scope.json` with matched repos

## EDU Portfolio Rollup

EDU AI metrics are aggregated from delivery team data:

- **AI-assisted PR Coverage**: Rebuilt from total AI PRs / total PRs across all teams (not an average of team percentages).
- **AI Active Developers**: Sum of unique AI-active developers across teams.

## Running

```bash
npm run pull:ai-adoption-metrics              # Incremental: from latest existing quarter onward
npm run pull:ai-adoption-metrics -- --full    # Full: re-fetch all quarters from scratch
```

By default the script reads the existing CSV, finds the latest quarter already present, and only re-fetches from that quarter onward (since it may have been in-progress). Historical data from earlier quarters is preserved. Pass `--full` to re-fetch everything from `currentYear - 1`.

Output lands in `backend/ai/generated/json_export_view.csv`.

## Adding a New Team Member

1. Add entry to `backend/ai/identity/team-user-map.json` with `team`, `name`, `githubLogin`, and optional `activeFrom`/`activeTo` dates.
2. Run `node scripts/ai/discover-repos.mjs` to refresh the repo mapping.
3. Run `npm run pull:ai-adoption-metrics` to include the member in metrics.

## Adding a New Org

Edit the `ORGS` array at the top of `scripts/ai/discover-repos.mjs` to include the new org name, then re-run discovery.
